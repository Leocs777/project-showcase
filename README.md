## Why This Repo Exists

I keep some production repositories private because they may include full app source, historical commits, local configuration, product data, unpublished implementation details, or App Store launch materials. This showcase gives a clean public view of the work without exposing sensitive code or private commit history.

## What Recruiters Can Review

- Product scope and user problem
- Technical stack and architecture decisions
- Privacy, compliance, and launch considerations
- Public URLs where available
- Implementation highlights and trade-offs
- What I owned end to end
